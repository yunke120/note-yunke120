表单

没有name属性无法提交数据

form属性

  • action 提交表单的地址
  • method 提交方式 [get, post] 必须设置表单元素name属性值，否则无法获取数据
  • target 打开窗口的方式 [_self, _blank]

Get请求与Post请求区别

  1. get请求时参数会跟在浏览器地址栏的后面，post请求不会(post请求会将数据存放在请求体中)
  2. get请求相对于post请求而言，安全性较低
  3. get请求传递的数据长度是有限的，而post请求基本没有限制(与服务器相关)
  4. get请求比post请求快（2倍左右）
  5. get请求有缓存（会将数据存放在浏览器中，即本地磁盘中），post请求无缓存

常用标签

  1. input
    • type: 元素的类型 [text, password, radio, checkbox, file， button, submit， reset, date, time]
      • radio:单选框通过设置name属性来分组
      file:文件域 表单需要设置enctype属性 enctype="multipart/form-data",提交方式为post
    • value: 元素的值
    • readonly: 只读状态
    • maxlength: 限制写入的长度
  2. textarea
    3. 定义可输入多行文本的控件
    • cols 文本的可见宽度
    • rows 可见高度
  3. label
    • for 当for属性与元素的id值一致时，点击lable标签，会自动向元素聚焦
  4. button
    5. 和input基本一样，不同的是button可以设置双标签
  5. select
    • multiple 设置下拉框可多选
    • option 设置下拉框选项标签
      • 属性：selected 默认项
      disabled 禁用项
      value 提交给服务器的选项值，如果设置value属性值，则提交的是value的值，如果未设置，则提交的是option中的双标签中的值
    • size 设置下拉框可见项数
    • disabled 禁用元素
